# UI优化提示词模板

## 使用说明

当需要优化某个页面时，使用以下模板向AI助手提供清晰的指令。这个模板基于任务管理页面的成功优化经验，确保全站UI风格统一。

## 标准提示词模板

```
请根据全站UI设计系统规范（@.cursor/rules/ui/ui-design-system.mdc），优化 @[页面文件路径] 页面UI，实现样式统一：

1. **主色调应用**：使用橘色（#FFA500）为主色调、白色（#FFFFFF）为背景色，遵循简洁大气原则
2. **字体规范**：不要在页面里写重复的 `style={{ fontFamily: ... }}`；字体应从 `web/src/app/globals.css body` 继承（如需等宽用 `am-mono`）
3. **表单输入框**：边框色都是指获得焦点时的颜色，优先使用全局类 `am-field`（等价于橙色 focus 边框与 ring）
4. **按钮样式**：
   - 主要按钮：`am-btn-primary`
   - 次要按钮：`am-btn-outline`
   - 重置按钮：`am-btn-reset`
   - 品牌幽灵按钮：`am-btn-ghost-brand`
5. **卡片样式**：使用 `am-card`
6. **搜索筛选区域**：使用 `am-filter-bar`
7. **注重留白与层次感**，确保视觉焦点明确，操作引导清晰

请确保：
- 所有用户可见文本应用无衬线字体
- 不要硬编码 `#FFA500/#FF8C00/...`，优先用 `brand.*` 色板或 `am-*` 公共类
- 所有按钮使用 `am-*` 规范类
- 所有输入框使用 `am-field`
- 间距和留白符合规范
- 响应式布局正确实现
```

## 简化版提示词（快速优化）

```
请根据 @.cursor/rules/ui/ui-design-system.mdc 规范优化 @[页面文件路径]，确保：
1. 主色调 #FFA500，白色背景
2. 所有文本使用无衬线字体
3. 输入框焦点边框为橙色
4. 按钮使用规范样式
5. 卡片和布局符合设计系统
```

## 提醒机制

### 方式一：在提示词中引用规范文件
每次优化时，在提示词开头添加：
```
@.cursor/rules/ui/ui-design-system.mdc
```
这样AI会自动读取规范文件，确保不偏离方向。

### 方式二：使用检查清单
优化完成后，对照规范文件中的"实现检查清单"逐项检查。

### 方式三：阶段性提醒
如果发现AI偏离了规范，可以这样提醒：
```
请回到 @.cursor/rules/ui/ui-design-system.mdc 规范，重新检查当前实现是否符合：
1. 颜色系统是否正确应用
2. 字体是否统一
3. 组件样式是否符合规范
```

## 常见问题处理

### 问题1：AI忘记了字体规范
**提醒方式**：
```
请确保所有文本都应用了无衬线字体，参考 @.cursor/rules/ui/ui-design-system.mdc 中的字体规范部分。
```

### 问题2：按钮颜色不一致
**提醒方式**：
```
请检查所有按钮是否符合 @.cursor/rules/ui/ui-design-system.mdc 中的按钮样式规范，主要按钮应使用 bg-[#FFA500]。
```

### 问题3：输入框焦点样式缺失
**提醒方式**：
```
请确保所有输入框、下拉框、文本域都应用了焦点样式：focus:border-[#FFA500] focus:ring-[#FFA500]/20
```

### 问题4：间距和留白不规范
**提醒方式**：
```
请参考 @.cursor/rules/ui/ui-design-system.mdc 中的"间距与布局"部分，确保间距符合规范。
```

## 优化流程建议

1. **准备阶段**
   - 确定要优化的页面文件
   - 查看当前页面存在的问题
   - 准备标准提示词

2. **执行阶段**
   - 使用标准提示词模板
   - 引用规范文件 `@.cursor/rules/ui/ui-design-system.mdc`
   - 让AI执行优化

3. **检查阶段**
   - 对照检查清单逐项检查
   - 测试响应式布局
   - 验证交互效果

4. **调整阶段**
   - 如有不符合规范的地方，使用提醒机制
   - 继续优化直到完全符合规范

## 示例：优化聊天页面

```
请根据全站UI设计系统规范（@.cursor/rules/ui/ui-design-system.mdc），优化 @web/src/app/chat/page.tsx 页面UI，实现样式统一：

1. **主色调应用**：使用橘色（#FFA500）为主色调、白色（#FFFFFF）为背景色
2. **字体规范**：所有文本使用无衬线字体
3. **表单输入框**：焦点时边框为橙色
4. **按钮样式**：使用规范的主要按钮、次要按钮样式
5. **卡片样式**：统一使用白色背景和浅灰色边框
6. **注重留白与层次感**，确保视觉焦点明确

请确保所有元素都符合 @.cursor/rules/ui/ui-design-system.mdc 中的规范。
```

## 注意事项

1. **始终引用规范文件**：在提示词中引用 `@.cursor/rules/ui/ui-design-system.mdc` 确保AI能读取最新规范
2. **具体指出问题**：如果发现不符合规范的地方，明确指出并引用相关规范章节
3. **逐步优化**：如果页面较大，可以分区域逐步优化，每次优化一个区域
4. **保持一致性**：确保同一页面的所有组件都使用相同的样式规范
